## New IPC commands

- `local_search(query: String) -> SearchResults` — in-process union search across catalog + enrichment + categories. Weighted scoring (name > friendly > category > summary > desc > tag), AND semantics across multi-term queries, capped at 200 hits split between formulae and casks. Replaces `brew_search` as the search store's default; `brew_search` + `brew_search_desc` remain registered for parity.
- `brew_upgrade_many(names: Vec<String>, ...)` — single batched `brew upgrade <a> <b> ...` invocation for curated upgrade selections. Validates names through the same allowlist regex as `brew_install`. Empty list → `InvalidArgument`.
- `update_relaunch(app)` — restarts the running app process via a 50ms-delayed `tauri::AppHandle::restart()` so the IPC response makes it back to the renderer before the process dies. Bridges the gap between the in-app updater's `.app.tar.gz` install and the user clicking "Relaunch now".

## Modified IPC commands

- `brew_list(state, force: Option<bool>)` — new `force` parameter bypasses `state.installed_cache` when true. Wired to the Dashboard Refresh button so `brew upgrade` runs from the user's terminal aren't masked by a stale cache indefinitely.

## User impact

- **v0.3.0 → v0.3.1 auto-update works.** The in-app updater (shipped in v0.3.0) is the canonical path; Settings → Network → Updates → Check now picks it up.
- **GitHub Keychain re-sign-in required once.** Bundle id rename means the old `dev.openbrew.browser` service ID is no longer reachable from the new binary. On first Star / Watch / File-issue interaction after upgrade, the existing Re-authorize toast button triggers Device Flow with the full scope set; ~3 clicks and they're back.
- **Search behavior changed.** No longer shells to `brew search`; the new local_search returns richer results from the same query. Some users may be surprised that previously-empty queries now return hits. The same query no longer needs to match brew's exact package name to return useful results.
- **Activity history may show more entries.** Cap raised from 50 to 200; existing localStorage stays valid.

## Notes

- The bundle id rename is irreversible at the user level (Keychain entries in the old service are orphaned). Worth doing now at 18 stars rather than later at 1800.
- `local_search` is the magic feature. Future iteration ideas: surface category-label match as a special "Browse → X" prompt at the top of results; weight installed packages higher; let users opt-out of fuzzy matching via Settings → Search → "Exact name only".
- One stable v0.3.x follow-up item: persist `last_checked_at` to disk so the auto-updater 24h floor honours typical morning/evening usage (currently in-memory; effectively never auto-fires across launches).
- The `weight::*` dead-code analyzer false-positive in `local_search` is a known Rust quirk with closures inside async fns. Worth a follow-up cleanup if/when the analyzer improves; not blocking.
