### Homebrew 6.x reality (real users blocked)
- **Tap-trust is now mandatory** in brew 6.0 — install needs `brew trust msitarzewski/brew-browser`

  (README + landing) updated to the 3-line form.
- **Tap cask was stale 0.5.1 + Apple-Silicon-only** → Intel users got "no cask" errors. Fixed in

  PRs #1 (cyberchen1995, merged) / #3 (kaalualu, closed-thanks) credited. **The tap cask must be
  version+sha256 bumped on EVERY release — it doesn't auto-follow GitHub releases.**

### Report-button brew-vs-app fix (PR #91)
- The ~50 `[brew-browser] X failed` issues were auto-generated by the in-app "Report to
  brew-browser" button on what are actually Homebrew failures. Fix: gate the report button on
  `exitCode` — brew ran & exited non-zero = Homebrew problem (show "run in Terminal", no report);
  no brew exit code (IPC/spawn) = real app bug (keep report). Tauri `ActivityDrawer`/`reportIssue.ts`
  + native `AppModel.isAppError` + `ActivityView`. (Distinct from `friendlyMessage`/`friendlify`,
  which is the curated KNOWN-error catalog incl. the sudo/admin-password trap.)

### Landing favicon (PR #93)
- Landing had no `/favicon.ico` (only an SVG link) → Google's favicon service (which the app's own
  `IconService` falls back to for cask icons) served a globe → brew-browser's own cask icon was the
  empty placeholder. Added a raster favicon set from `brew-browser.svg`; deployed to the host
  (rsync NO --delete — the web root also serves updater.json/appcast/native zips).
